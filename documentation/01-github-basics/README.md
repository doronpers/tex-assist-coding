# GitHub Basics

Welcome! If you're new to GitHub, you're in the right place. This section will help you understand what GitHub is, why it matters, and how to use it confidently.

## What You'll Learn Here

- What GitHub actually is (and why developers love it)
- How to navigate repositories without fear
- Understanding branches, commits, and pull requests
- Working with others safely
- Recovering when things go wrong

## Why GitHub Matters

Think of GitHub as:
- **Google Docs for code** - Multiple people can work together
- **Time machine** - See any previous version of your code
- **Portfolio** - Show off what you've built
- **Community** - Learn from millions of open source projects

- Some people understand it in an afternoon ‚Üí That's one path

- **Your pace is YOUR pace** - compare only to your past self

**Consider your micro-motives:**
- If you love understanding systems ‚Üí Deep dive into Git internals
- If you love practical tools ‚Üí Just learn what you need for your project
- If you love visual thinking ‚Üí Use GitHub Desktop or GitKraken
- If you love command line ‚Üí Master git commands
- If you love collaboration ‚Üí Focus on PRs and issues
- If you work solo ‚Üí Basic commit/push might be enough

**Permission: Learn GitHub YOUR way. Skip what doesn't serve you right now.**

## Feeling Stuck?

**If GitHub feels overwhelming:**
- Remember: EVERYONE finds it confusing at first
- GitHub has been around since 2008 and has accumulated a lot of features
- You only need to understand a tiny fraction to be productive
- Ask AI: "Explain GitHub like I'm 5 years old"

**If you make a mistake:**
- That's literally what Git was designed for - fixing mistakes
- Almost everything is reversible
- See [When Things Break](when-things-break.md) for recovery strategies

**Still confused after reading?**
- Try it anyway - clicking around won't break anything
- Come back and re-read after trying
- Ask a human (no shame!)
- Open an issue in this repo
